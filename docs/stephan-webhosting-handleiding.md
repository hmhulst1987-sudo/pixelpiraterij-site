# Stephan's Handleiding: webhosting, websites en het verkoopverhaal

Deze handleiding is voor iemand die goed kan verkopen, maar nog niet technisch is in websites, hosting en domeinen.

Doel:
- snappen wat de basisbegrippen betekenen
- het in gewone taal kunnen uitleggen
- er geloofwaardig over kunnen praten met klanten
- niet gaan bluffen over dingen die je nog niet volledig snapt

Belangrijkste regel:

Je hoeft niet alles technisch te kunnen bouwen om het goed te verkopen.
Je moet vooral snappen:
- wat iets ongeveer doet
- waarom het belangrijk is
- welk probleem het voor de klant oplost

## 1. De simpelste uitleg van allemaal

Als iemand vraagt wat er nodig is voor een website, kun je het zo zien:

- De domeinnaam is het adres
- De hosting is het huis of het stuk grond waar de website staat
- DNS is de routering die mensen naar het juiste huis stuurt
- De website zelf is de inrichting en inhoud van het huis
- Cloudflare is de beveiliger en verkeersregelaar voor de deur
- Google Cloud is de technische infrastructuur achter de schermen
- SEO helpt mensen de website vinden via Google
- Onderhoud is zorgen dat alles blijft werken, veilig blijft en snel blijft

## 2. De basisbegrippen in Jip en Janneke taal

### Wat is een domeinnaam?

Voorbeeld: `pixelpiraterij.nl`

In gewone taal:
Een domeinnaam is het webadres dat iemand intypt om een site te bezoeken.

Vergelijking:
Het is het straatadres van een pand.

Wat je tegen een klant kunt zeggen:
`De domeinnaam is gewoon jouw online adres. Daarmee kunnen mensen je vinden en onthouden.`

Wat je beter niet zegt:
`Dat regelen we wel ergens in de code.`

### Wat is hosting?

In gewone taal:
Hosting is de plek waar de website echt draait en opgeslagen staat.

Vergelijking:
Als je domeinnaam het adres is, dan is hosting het gebouw of de grond waarop alles staat.

Wat je tegen een klant kunt zeggen:
`Je website moet ergens wonen. Hosting is die plek. Zonder hosting heb je wel een naam, maar geen werkende site.`

Wat hosting meestal omvat:
- opslag van bestanden
- servercapaciteit
- snelheid
- beveiliging
- bereikbaarheid
- soms e-mail of databases

### Wat is een server?

In gewone taal:
Een server is een computer die dag en nacht klaarstaat om websitebezoekers te helpen.

Wat je tegen een klant kunt zeggen:
`Een server is de motorruimte achter de site. Die zorgt dat bezoekers de site snel en stabiel zien.`

### Wat is DNS?

DNS staat voor Domain Name System.

In gewone taal:
DNS zorgt dat een domeinnaam wordt vertaald naar de juiste technische plek.

Vergelijking:
Jij zegt `pixelpiraterij.nl`, DNS zegt: `oké, die moet naar die server daar`.

Wat je tegen een klant kunt zeggen:
`DNS is de schakel tussen de domeinnaam en de plek waar de website echt staat.`

### Wat zijn nameservers?

In gewone taal:
Nameservers bepalen wie de baas is over de DNS-instellingen van een domein.

Vergelijking:
Ze zijn een soort hoofdkantoor waar wordt besloten hoe het verkeer wordt doorgestuurd.

Wat je tegen een klant kunt zeggen:
`Nameservers bepalen waar het domein zijn route-instructies ophaalt.`

### Wat is een A-record?

In gewone taal:
Een A-record koppelt een domeinnaam aan een IP-adres.

Vergelijking:
Het is alsof je zegt: dit adres hoort bij dit gebouw.

Je hoeft dit niet uit je hoofd uit te leggen aan klanten, maar wel ongeveer te herkennen.

### Wat is een CNAME?

In gewone taal:
Een CNAME laat het ene webadres verwijzen naar een ander adres.

Voorbeeld:
`www.domein.nl` mag doorverwijzen naar `domein.nl`

### Wat is SSL?

Je ziet dat als het slotje in de browser en `https`.

In gewone taal:
SSL zorgt voor een beveiligde verbinding tussen bezoeker en website.

Wat je tegen een klant kunt zeggen:
`Dat slotje laat zien dat gegevens versleuteld worden verstuurd. Dat is belangrijk voor vertrouwen en veiligheid.`

## 3. Wat doet Cloudflare?

In gewone taal:
Cloudflare zit tussen de bezoeker en de website in.

Cloudflare helpt vaak met:
- beveiliging
- snelheid
- caching
- DNS
- bescherming tegen aanvallen
- SSL

Vergelijking:
Cloudflare is de portier, beveiliger en verkeersregelaar tegelijk.

Wat je tegen een klant kunt zeggen:
`Cloudflare helpt om de site sneller, veiliger en stabieler te maken.`

Wat Cloudflare niet is:
- niet automatisch de website zelf
- niet per se de plek waar alles gebouwd wordt
- niet hetzelfde als Google Cloud

## 4. Wat is Google Cloud?

In gewone taal:
Google Cloud is een groot platform waar je technische onderdelen van websites en apps kunt draaien.

Denk aan:
- servers
- databases
- opslag
- apps
- backend-systemen

Vergelijking:
Google Cloud is een groot bedrijventerrein met allemaal technische gebouwen en diensten.

Wat je tegen een klant kunt zeggen:
`Google Cloud is de professionele infrastructuur waar je website of app stabiel op kan draaien.`

## 5. Wat is Cloud Run?

In gewone taal:
Cloud Run is een dienst binnen Google Cloud die stukjes software of een website kan draaien zonder dat je zelf een hele server hoeft te beheren.

Vergelijking:
In plaats van zelf een heel pand te kopen en te onderhouden, huur je een slimme ruimte die automatisch meebeweegt als er meer of minder bezoekers zijn.

Waarom het fijn is:
- schaalbaar
- je betaalt vaak slimmer
- minder gedoe met serverbeheer

Wat je tegen een klant kunt zeggen:
`Cloud Run is handig als je iets moderns en schaalbaars wilt draaien zonder ouderwets servergedoe.`

## 6. Wat is SEO?

SEO staat voor Search Engine Optimization.

In gewone taal:
SEO is alles wat helpt om beter gevonden te worden in Google.

Dat gaat bijvoorbeeld over:
- goede teksten
- duidelijke pagina-opbouw
- juiste titels
- snelheid
- mobielvriendelijkheid
- betrouwbaarheid

Belangrijk:
SEO is niet alleen een trucje. Het is vooral duidelijkheid, kwaliteit en relevantie.

Wat je tegen een klant kunt zeggen:
`SEO helpt Google begrijpen waar je website over gaat, zodat je beter gevonden kunt worden door de juiste mensen.`

Wat je beter niet belooft:
- `Wij zetten je binnen twee weken op plek 1 in Google`

Beter:
`We zorgen dat de technische basis, structuur en inhoud goed staan, zodat je site beter kan presteren in zoekmachines.`

## 7. Wat is caching?

In gewone taal:
Caching betekent dat onderdelen van de website tijdelijk slim worden bewaard, zodat ze sneller geladen worden.

Vergelijking:
Alsof iemand de meest gebruikte spullen alvast binnen handbereik legt.

Wat je tegen een klant kunt zeggen:
`Caching helpt om een website sneller te laden en de server minder onnodig werk te laten doen.`

## 8. Wat is uptime?

In gewone taal:
Uptime betekent hoe vaak een website gewoon bereikbaar is.

Voorbeeld:
Als een site bijna altijd online is, heeft die hoge uptime.

Wat je tegen een klant kunt zeggen:
`Uptime gaat over betrouwbaarheid. Een website heeft pas waarde als hij ook echt bereikbaar blijft.`

## 9. Wat zijn onderhoud en beheer?

In gewone taal:
Een website online zetten is niet het einde. Daarna begint het gezond houden pas echt.

Onderhoud kan betekenen:
- updates
- foutjes oplossen
- backups
- snelheid in de gaten houden
- beveiliging checken
- formulieren testen
- domein en DNS bewaken

Wat je tegen een klant kunt zeggen:
`Een website is geen flyer die je eenmalig drukt. Het is eerder een winkelpand dat netjes, veilig en werkend moet blijven.`

## 10. Wat is een backup?

In gewone taal:
Een backup is een reservekopie van de website of belangrijke gegevens.

Waarom belangrijk:
Als er iets stukgaat, gehackt wordt of per ongeluk verdwijnt, kun je herstellen.

Wat je tegen een klant kunt zeggen:
`Een backup is je vangnet. Je hoopt hem niet nodig te hebben, maar je bent heel blij als hij er is.`

## 11. Wat is e-mailhosting of e-mail forwarding?

In gewone taal:
Dat gaat over e-mailadressen op je domein, zoals `info@bedrijfsnaam.nl`.

Dat kan bijvoorbeeld:
- een echte mailbox zijn
- of alleen een doorstuuradres naar Gmail of iets anders

Wat je tegen een klant kunt zeggen:
`We kunnen zorgen dat je een professioneel e-mailadres op je eigen domein gebruikt.`

## 12. Het verschil tussen webdesign, hosting en beheer

### Webdesign

Dit is:
- hoe de site eruitziet
- hoe hij voelt
- hoe de structuur en inhoud worden opgebouwd

### Hosting

Dit is:
- waar hij draait
- hoe snel en stabiel hij is
- hoe hij technisch bereikbaar blijft

### Beheer

Dit is:
- het doorlopend gezond houden
- aanpassen
- beveiligen
- monitoren

Kort gezegd:
- design is de voorkant
- hosting is de technische basis
- beheer is de nazorg en continuiteit

## 13. Wat verkoop je nou eigenlijk echt aan een klant?

Niet:
- pixels
- code
- een hip thema

Wel:
- vertrouwen
- duidelijkheid
- bereikbaarheid
- professionaliteit
- online geloofwaardigheid
- rust

Een klant koopt meestal niet `hosting`.
Een klant koopt:
- een site die werkt
- een partij die hem helpt
- minder stress
- een professionelere uitstraling
- een betere eerste indruk

## 14. Hoe Stephan het geloofwaardig kan uitleggen

### De simpele versie

`Een website bestaat grofweg uit drie dingen: je domeinnaam, de hosting en de site zelf. De domeinnaam is je online adres, hosting is de plek waar de site draait, en de site zelf is wat mensen uiteindelijk zien en gebruiken. Daarom kijken we niet alleen naar design, maar ook naar snelheid, veiligheid en bereikbaarheid.`

### De iets sterkere versie

`Wij kijken niet alleen naar hoe een website eruitziet, maar ook naar wat er technisch achter moet staan om hem stabiel, snel en veilig te laten werken. Denk aan domein, DNS, hosting, beveiliging, Cloudflare en onderhoud. Dat zijn de dingen die bepalen of een site ook echt professioneel draait.`

### De verkoopbare versie

`Een mooie website is leuk, maar als hij traag is, slecht gevonden wordt of technisch rommelig staat, kost dat uiteindelijk vertrouwen. Daarom verkopen wij niet alleen een design, maar een complete online basis die er goed uitziet en ook echt goed functioneert.`

## 15. Wat hij wel en niet moet zeggen

### Wel zeggen

- `Dat is de technische route achter de website`
- `Dat regelen we in de hosting en DNS-laag`
- `Daar zorgen we voor in de beveiliging en bereikbaarheid`
- `Dat valt onder beheer en onderhoud`
- `Dat is belangrijk voor snelheid, stabiliteit en vertrouwen`
- `Ik kan het technisch nog specifieker voor je laten uitschrijven als je wilt`

### Niet zeggen

- `Oh dat is gewoon een knopje`
- `Dat stelt niets voor`
- `Dat doen we wel even`
- `Geen idee maar dat komt goed`
- technische termen droppen die hij zelf niet kan uitleggen

Beste alternatief als hij iets niet weet:

`Ik wil je daar netjes op antwoorden, dus ik check dat even technisch voor je en kom er strak op terug.`

Dat klinkt professioneel, niet zwak.

## 16. Veelgestelde klantvragen met goede antwoorden

### Waarom heb ik hosting nodig?

`Omdat je website ergens moet draaien. Zonder hosting heb je wel een domeinnaam, maar geen werkende website.`

### Waarom kan ik niet gewoon een goedkoop pakketje ergens nemen?

`Dat kan wel, maar goedkoop betekent niet automatisch passend. Het gaat om snelheid, betrouwbaarheid, ondersteuning, veiligheid en hoeveel controle je hebt.`

### Waarom is onderhoud nodig als de site al online staat?

`Omdat online staan niet hetzelfde is als gezond blijven. Net als bij een auto of pand moet je blijven controleren, updaten en beveiligen.`

### Wat doet Cloudflare voor mij?

`Cloudflare helpt met snelheid, beveiliging en stabiliteit. Het is een extra beschermlaag tussen de bezoeker en de website.`

### Wat betekent SEO precies?

`SEO helpt zoekmachines begrijpen wat jouw site doet, zodat je beter gevonden kunt worden door de juiste mensen.`

### Wat is het verschil tussen Google Cloud en Cloudflare?

`Google Cloud is vooral infrastructuur waar dingen op draaien. Cloudflare zit meer aan de voorkant als bescherm- en versnellingslaag.`

## 18. Hoe verkoop je een website zonder technisch geneuzel

Niet beginnen met:
- servertypes
- records
- frameworks
- code

Wel beginnen met:
- wat wil de klant bereiken
- wat moet de site uitstralen
- wat moet de site opleveren
- hoeveel rust of support de klant nodig heeft

Slimme volgorde in een verkoopgesprek:

1. Wat moet de website voor je doen?
2. Hoe professioneel moet het overkomen?
3. Heb je al een domeinnaam?
4. Heb je al hosting of draait het nu ergens?
5. Wil je vooral ontzorgd worden, of wil je zelf veel beheren?

Dan kun je veel makkelijker uitleggen waarom hosting, beheer en technische keuzes belangrijk zijn.
